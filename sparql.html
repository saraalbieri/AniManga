Validazione tramite SPARQL Queries

Le query SPARQL sono state usate prima per esplorare il dominio anime/manga in Wikidata e solo successivamente per verificare la proprietà wdt:P9071. In questo modo il gap sugli archetipi narrativi non è stato assunto in partenza, ma individuato progressivamente confrontando le proprietà disponibili sui personaggi.

Fase esplorativa 1: trovare le opere su Wikidata

La prima operazione è stata cercare le entità Wikidata relative alle tre opere scelte: One Piece, Naruto e Death Note. Questa query serve a verificare che il dominio sia già rappresentato nel Knowledge Graph.

PREFIX wd: <http://www.wikidata.org/entity/>
PREFIX schema: <http://schema.org/>
PREFIX wikibase: <http://wikiba.se/ontology#>
PREFIX bd: <http://www.bigdata.com/rdf#>

SELECT DISTINCT ?work ?workLabel ?description
WHERE {
  VALUES ?work { wd:Q710324 wd:Q25929253 wd:Q718624 }
  OPTIONAL { ?work schema:description ?description . FILTER(LANG(?description) = "en") }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en,it". }
}
ORDER BY ?workLabel

Perché è utile: mostra che le opere sono già presenti in Wikidata e possono essere usate come punto di partenza per esplorare i personaggi collegati.

Fase esplorativa 2: recuperare i personaggi collegati alle opere

Dopo aver individuato le opere, abbiamo cercato i personaggi collegati. La query usa UNION perché Wikidata può rappresentare il legame personaggio-opera con proprietà diverse, come P1441 e P1080, e filtra i risultati al tipo anime character.

PREFIX wd: <http://www.wikidata.org/entity/>
PREFIX wdt: <http://www.wikidata.org/prop/direct/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?character ?characterLabel ?work ?workLabel
WHERE {
  VALUES ?work { wd:Q710324 wd:Q25929253 wd:Q718624 }
  { ?character wdt:P1441 ?work . }
  UNION
  { ?character wdt:P1080 ?work . }
  ?work rdfs:label ?workLabel .
  ?character rdfs:label ?characterLabel .
  ?character wdt:P31 wd:Q80447738 . # anime character
  FILTER(LANG(?workLabel) = "en")
  FILTER(LANG(?characterLabel) = "en")
}
ORDER BY ?workLabel ?characterLabel
LIMIT 100

Perché è utile: questa query ci ha permesso di vedere che Wikidata contiene già numerosi personaggi delle tre opere, quindi il problema non è l'assenza delle entità.

Dal confronto delle proprietà al gap

Dopo aver confrontato personaggi principali e secondari, abbiamo cercato una proprietà adatta a rappresentare gli archetipi narrativi. La proprietà individuata è wdt:P9071, character type. Il passaggio successivo è stato quindi verificare se questa proprietà fosse effettivamente valorizzata nei personaggi analizzati.

Conclusione della fase esplorativa: il vocabolario esiste, ma il dato manca o è incompleto. Questo è il gap semantico su cui si basa il progetto.

Query 1: personaggi selezionati e valori P9071

Questa query usa i QID verificati dei 15 personaggi e controlla, tramite OPTIONAL, se esiste un valore per wdt:P9071.

PREFIX wd: <http://www.wikidata.org/entity/>
PREFIX wdt: <http://www.wikidata.org/prop/direct/>
PREFIX wikibase: <http://wikiba.se/ontology#>
PREFIX bd: <http://www.bigdata.com/rdf#>

SELECT ?anime ?animeLabel ?character ?characterLabel ?characterType ?characterTypeLabel
WHERE {
  VALUES ?anime { wd:Q710324 wd:Q25929253 wd:Q718624 }
  VALUES ?character { wd:Q477948 wd:Q858432 wd:Q877964 wd:Q1061765 wd:Q1190653 wd:Q931 wd:Q1740 wd:Q203240 wd:Q1767 wd:Q1043344 wd:Q843545 wd:Q52863 wd:Q687038 wd:Q2328583 wd:Q138008 }
  { ?character wdt:P1441 ?anime . }
  UNION
  { ?character wdt:P1080 ?anime . }
  OPTIONAL { ?character wdt:P9071 ?characterType . }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en,it". }
}
ORDER BY ?animeLabel ?characterLabel ?characterTypeLabel

Query 2: personaggi collegati ai tre anime

Questa query usa UNION perché Wikidata può collegare i personaggi alle opere attraverso proprietà diverse. Le tre opere sono selezionate tramite QID verificati e i risultati sono limitati alle entità dichiarate come anime character, così evitiamo ambiguità dovute alle label.

PREFIX wd: <http://www.wikidata.org/entity/>
PREFIX wdt: <http://www.wikidata.org/prop/direct/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?character ?characterLabel ?work ?workLabel
WHERE {
  VALUES ?work { wd:Q710324 wd:Q25929253 wd:Q718624 }
  { ?character wdt:P1441 ?work . }

  { ?character wdt:P1080 ?work . }
  ?work rdfs:label ?workLabel .
  ?character rdfs:label ?characterLabel .
  ?character wdt:P31 wd:Q80447738 . # anime character
  FILTER(LANG(?workLabel) = "en")
  FILTER(LANG(?characterLabel) = "en")
}
ORDER BY ?workLabel ?characterLabel
LIMIT 100

Query 4: stock character come vocabolario per l'IA

Questa query estrae da Wikidata gli archetipi classificati come stock character (wd:Q636497). Questo insieme viene usato come vocabolario controllato da fornire all'intelligenza artificiale: il modello non sceglie liberamente un archetipo, ma confronta il personaggio con una lista di candidati già presenti in Wikidata.

PREFIX wd: <http://www.wikidata.org/entity/>
PREFIX wdt: <http://www.wikidata.org/prop/direct/>
PREFIX schema: <http://schema.org/>
PREFIX wikibase: <http://wikiba.se/ontology#>
PREFIX bd: <http://www.bigdata.com/rdf#>

SELECT ?stockCharacter ?stockCharacterLabel ?description
WHERE {
  ?stockCharacter wdt:P31 wd:Q636497 . # stock character
  OPTIONAL {
    ?stockCharacter schema:description ?description .
    FILTER(LANG(?description) = "en")
  }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en,it". }
}
ORDER BY ?stockCharacterLabel

Per ogni candidato vengono recuperati QID, label e descrizione. La descrizione è importante perché permette all'IA di motivare la scelta confrontando i tratti del personaggio con il significato dell'archetipo.

Query 4b: contesto Wikidata per il prompt LLM

Questa query estrae il contesto strutturato che viene fornito al modello LLM prima della scelta dell'archetipo. Le proprietà selezionate descrivono appartenenza, ruolo e caratterizzazione del personaggio, rendendo la classificazione più motivata e meno dipendente dalla sola label.

PREFIX wd: <http://www.wikidata.org/entity/>
PREFIX wdt: <http://www.wikidata.org/prop/direct/>
PREFIX schema: <http://schema.org/>
PREFIX wikibase: <http://wikiba.se/ontology#>
PREFIX bd: <http://www.bigdata.com/rdf#>

SELECT ?character ?characterLabel ?description ?instanceOfLabel ?workLabel ?universeLabel ?memberOfLabel ?occupationLabel ?creatorLabel ?genderLabel ?characterTypeLabel ?personalityDatabaseId
WHERE {
  VALUES ?character { wd:Q1740 } # esempio: Sasuke Uchiha
  OPTIONAL {
    ?character schema:description ?description .
    FILTER(LANG(?description) = "en")
  }
  OPTIONAL { ?character wdt:P31 ?instanceOf . }
  OPTIONAL { ?character wdt:P1441 ?work . }
  OPTIONAL { ?character wdt:P1080 ?universe . }
  OPTIONAL { ?character wdt:P463 ?memberOf . }
  OPTIONAL { ?character wdt:P106 ?occupation . }
  OPTIONAL { ?character wdt:P170 ?creator . }
  OPTIONAL { ?character wdt:P21 ?gender . }
  OPTIONAL { ?character wdt:P9071 ?characterType . }
  OPTIONAL { ?character wdt:P10757 ?personalityDatabaseId . }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en,it". }
}
LIMIT 20
